# PRD: Architecture of Life 執筆自動化システム

## 1. プロジェクト概要

### 1.1 背景
「Architecture of Life（人生の設計図）」は、27歳でUIUXデザイナー・PdM・古民家ホテル経営者として活動するKentoが、キャリア戦略・資産形成・クリエイティブビジネスの知見を発信するNoteマガジンプロジェクト。

### 1.2 目的
- 100記事のバックログを効率的に執筆・公開する
- AIを最大限活用し、週6時間で週3記事の執筆ペースを実現
- X（Twitter）連携による認知拡大の自動化

### 1.3 ターゲット読者
1. **過去の自分（23歳）**: 現場に絶望している建築・クリエイティブ志望者
2. **3年前の自分（25歳）**: IT転身後、ロールモデル不在で焦る若手
3. **潜在層**: 会社員を辞めずに「自由」を手に入れたいハイブリッド志向層

### 3.3 CLAUDE.md（Claude Code用コンテキスト）の構成
```markdown
# Architecture of Life - Claude Code Context

## 著者について
- 27歳、UIUXデザイナー・PdM（リクルート）、株式会社tokoro代表
- 建築→IT転職、タワマン購入、古民家ホテル経営
- 価値観：自由、精神的安定、評価されるものづくり

## 文体ガイドライン
- 一人称：「私」
- 語り口：親しみやすいが、ロジカル。感情論ではなく構造で語る

## 記事構成の型
1. Pain（読者の痛み）を最初に提示
2. 自分の体験を交えたストーリー
3. Solution（構造的な解決策）

## 参照すべきファイル
- context/profile.md: 詳細プロフィール
- context/voice-guide.md: 文体詳細
- backlog/article-backlog.md: 記事一覧
```

## 4. ワークフロー設計

### 4.1 週次執筆サイクル（週3記事 × 6時間）

```
┌─────────────────────────────────────────────────────────────┐
│  Day 1（2時間）: 記事A                                        │
│  ├─ [10分] Claude Codeで下書き生成                           │
│  ├─ [60分] 一次情報追加・校閲・編集                          │
│  ├─ [20分] X投稿リスト生成（自動）→ 確認・調整               │
│  └─ [30分] Note投稿・フォーマット調整                        │
├─────────────────────────────────────────────────────────────┤
│  Day 3（2時間）: 記事B                                        │
│  └─ 同上                                                     │
├─────────────────────────────────────────────────────────────┤
│  Day 5（2時間）: 記事C                                        │
│  └─ 同上                                                     │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 記事生成フロー（詳細）

```
[Step 1] 記事選定
    │
    ├─ GitHub Issuesから次の記事を選択（P1 → P2 → P3の順）
    │
    ▼
[Step 2] 下書き生成（Claude Code）
    │
    ├─ コマンド: claude "記事001の下書きを生成して"
    ├─ 参照: CLAUDE.md + context/ + バックログの該当記事
    ├─ 出力: articles/magazine-X/XXX/draft.md
    │
    ▼
[Step 3] 一次情報追加・編集（人間）
    │
    ├─ 具体的なエピソード、数字、固有名詞を追加
    ├─ 事実確認、トーン調整
    │
    ▼
[Step 4] 校閲（Claude Code）
    │
    ├─ コマンド: claude "この記事を校閲して"
    ├─ チェック項目: 誤字脱字、論理矛盾、文体の一貫性
    │
    ▼
[Step 5] X投稿リスト生成（Claude Code）
    │
    ├─ コマンド: claude "この記事からX投稿を10個抽出して"
    ├─ 出力: articles/magazine-X/XXX/x-posts.md
    │
    ▼
[Step 6] 公開
    │
    ├─ Note: final.mdをコピペ → フォーマット調整 → 公開
    ├─ X: x-posts.mdから1日2-3投稿をスケジュール
    │
    ▼
[Step 7] 記録
    │
    └─ meta.ymlに公開日、URL、初期反応を記録
```

### 4.3 X投稿の運用フロー

```
記事公開 → X投稿リスト（10個/記事）生成
                ↓
        週30投稿分のストック
                ↓
        1日2-3投稿 × 週7日 = 14-21投稿/週
                ↓
        約1.5週間分のバッファを常に確保
```

**X投稿の型**
1. **刺さるワンライナー**: 記事内の印象的な一文
2. **数字フック**: 「27歳で年収2000万」「稼働率70%」など
3. **問いかけ**: 「〜と思っていませんか？」
4. **逆張り**: 「〜は間違いだ」
5. **記事リンク付きまとめ**: 公開直後用

## 5. プロンプトテンプレート

### 5.1 下書き生成プロンプト
```markdown
# 記事下書き生成

## 対象記事
- 記事番号: {番号}
- タイトル: {タイトル}
- Pain: {読者の痛み}
- Solution: {解決策の方向性}

## 指示
1. context/profile.md と context/voice-guide.md を参照
2. 以下の構成で3000-4000字の記事を生成:
   - リード文（Pain提示）: 200字
   - 導入（自分の体験への橋渡し）: 300字
   - 本文（ストーリー + 構造的解説）: 2000-2500字
   - まとめ（Solution + CTA）: 500字
3. 【要追記】マークで一次情報が必要な箇所を明示

## 出力先
articles/magazine-{X}/{記事ID}/draft.md
```

### 5.2 X投稿抽出プロンプト
```markdown
# X投稿リスト生成

## 対象記事
{記事のfinal.mdの内容}

## 指示
以下の5タイプで各2投稿、計10投稿を生成:
1. 刺さるワンライナー（記事内の印象的な一文を140字以内に）
2. 数字フック（具体的な数字を含む投稿）
3. 問いかけ（読者への質問形式）
4. 逆張り（常識への反論）
5. 記事リンク付きまとめ（記事URLを含む）

## フォーマット
各投稿は以下の形式で出力:
---
[タイプ] 刺さるワンライナー
[投稿文]
XXXXXXXXXXXXXXX
[投稿予定日] （空欄）
---

## 出力先
articles/magazine-{X}/{記事ID}/x-posts.md
```

## 7. リスクと対策

| リスク | 影響度 | 対策 |
|--------|--------|------|
| AI生成感が出る | 高 | voice-guide.mdを詳細化、校閲プロセスの強化 |
| 一次情報の追加が重い | 中 | 記事ごとに「追記ポイント」を3箇所以内に絞る |
| X投稿がバズらない | 低 | A/Bテスト（投稿時間、フック）、初期は実験と割り切る |
| 週3本ペースが維持できない | 中 | バッファ記事を2本ストック、P3記事は軽量化 |

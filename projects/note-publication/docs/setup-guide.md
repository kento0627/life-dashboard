## 1. Claude Codeとは？（初心者向け）

### 1.1 概要
Claude Codeは、ターミナル（コマンドライン）からClaudeを使えるツールです。普段のClaude.aiとの違いは：

| 項目 | Claude.ai（Web） | Claude Code |
|------|------------------|-------------|
| 操作場所 | ブラウザ | ターミナル |
| ファイル操作 | 手動アップロード | 直接読み書き可能 |
| コンテキスト | 毎回説明が必要 | CLAUDE.mdを自動で読む |
| 向いている用途 | 単発の質問 | プロジェクト単位の作業 |

### 1.2 なぜ執筆にClaude Codeが向いているか

```
従来のやり方（Claude.ai）:
┌─────────────────────────────────────────┐
│ 毎回チャットで...                        │
│ 「私は27歳で、建築からITに転職して...」  │
│ 「文体はこんな感じで...」                │
│ 「この記事を書いて...」                  │
└─────────────────────────────────────────┘
        ↓ 毎回同じ説明の繰り返し...

Claude Codeのやり方:
┌─────────────────────────────────────────┐
│ CLAUDE.md に全部書いておく               │
│        ↓                                │
│ ターミナルで一言:                        │
│ $ claude "記事001の下書きを書いて"       │
│        ↓                                │
│ 自動でコンテキストを理解して生成         │
└─────────────────────────────────────────┘
```

### 1.3 インストール方法

```bash
# 1. Node.jsがインストールされていることを確認
node --version  # v18以上が必要

# 2. Claude Codeをインストール
npm install -g @anthropic-ai/claude-code

# 3. APIキーを設定（Anthropicのダッシュボードで取得）
export ANTHROPIC_API_KEY="sk-ant-xxxxx"

# 4. 動作確認
claude --version
```

### 1.4 基本的な使い方

```bash
# プロジェクトフォルダに移動
cd ~/projects/architecture-of-life

# Claudeに話しかける（CLAUDE.mdを自動で読み込む）
claude "記事001の下書きを生成して"

# ファイルを指定して作業
claude "articles/001/draft.md を校閲して"

# 対話モードで起動（複数回やり取りしたい時）
claude
> 記事001について相談したい
> ...
```

## 2. リポジトリ構成（モダン版）

### 2.1 ディレクトリ構造

```
architecture-of-life/
│
├── CLAUDE.md                      # メインの指示書（最重要）
│
├── .claude/                       # Claude Code用の設定フォルダ
│   ├── rules/                     # ルール分割管理
│   │   ├── writing-style.md       # 文体ルール
│   │   ├── article-structure.md   # 記事構成ルール
│   │   └── x-posts.md             # X投稿ルール
│   │
│   └── skills/                    # スキル（再利用可能なプロンプト）
│       ├── generate-draft.md      # 下書き生成スキル
│       ├── review-article.md      # 校閲スキル
│       ├── extract-x-posts.md     # X投稿抽出スキル
│       └── brainstorm.md          # アイデア出しスキル
│
├── context/                       # 著者情報・背景
│   ├── profile.md                 # 詳細プロフィール
│   ├── story-bank.md              # エピソード・体験談DB
│   └── keywords.md                # 頻出キーワード・用語集
│
├── articles/                      # 記事本体
│   ├── _template/                 # 記事テンプレート
│   │   ├── draft.md
│   │   ├── final.md
│   │   ├── x-posts.md
│   │   └── meta.yml
│   │
│   ├── 001-industry-despair/      # 各記事フォルダ
│   ├── 002-pivoting-career/
│   └── ...
│
├── backlog/                       # 企画管理
│   └── article-ideas.md           # 100記事バックログ
│
└── docs/                          # ドキュメント
    ├── setup-guide.md             # このファイル
    └── workflow.md                # 運用フロー
```

### 2.2 なぜこの構成か？

**`.claude/rules/`** - ルールを分割管理
- CLAUDE.mdが長くなりすぎると読みにくい
- ルールをカテゴリ別に分割して管理
- Claude Codeは `.claude/rules/` 内のファイルも自動で読む

**`.claude/skills/`** - 再利用可能なプロンプト
- 「下書き生成」「校閲」などの定型作業をテンプレート化
- コマンド一発で呼び出せる
- 改善したら全記事に反映される

**`context/story-bank.md`** - エピソードDB
- 一次情報（体験談）を蓄積
- AIが適切なエピソードを引用できる
- インスタ投稿などの過去コンテンツも格納

### 3.2 .claude/rules/writing-style.md（文体ルール）

```markdown
# 文体ルール（Voice Guide）

## 基本トーン
- 親しみやすく、でもロジカル
- 感情論ではなく「構造」で語る
- 自虐的なユーモアを交える
- 読者と同じ目線（上から目線にならない）

## 一人称・語尾
- 一人称: 「僕」
- 語尾: カジュアル（「〜だよ」「〜なんだけど」「〜だと思う」）
- 笑いの表現: 「w」「笑」は適度に使用OK
- 若者言葉: 「マジ」「サイコー」など適度にOK

## 文章の特徴
- 結論を先に述べる
- 具体的な数字を出す（「26歳」「一泊8万円」「関東全ての自治体」）
- 問いかけを投げかける（「〜ってなくない？」）
- 体験ベースで語る（抽象論を避ける）

## 禁止事項
- 過度な自慢、マウント
- 抽象的な精神論（「頑張れば夢は叶う」的な）
- 上から目線のアドバイス
- 堅すぎるビジネス文体（「〜でございます」「〜いたします」）

## 参考フレーズ（実際の投稿から）
- 「結論、〜した。（再現性は不明）」
- 「じゃあ何があったか？何となくの自信！w それだけw」
- 「振り返ると、全く絶望はなかった。」
- 「いやサイコーかよ！」
- 「これは、〜していたら絶対になかった未来だと思う。」
- 「何で〜か？そして何で〜か？」

## 文章構成パターン
1. 結論・主張をバン！と出す
2. 「じゃあどうしたか？」で展開
3. 具体的なエピソード
4. 学び・気づきをまとめる
5. 読者への問いかけで締める
```

### 3.3 .claude/rules/article-structure.md（記事構成ルール）

```markdown
# 記事構成ルール

## 基本構成（3000-4000字目安）

### 1. リード文（200-300字）
- 読者の「Pain（痛み）」を突く
- 「〜と思っていませんか？」「〜で悩んでいませんか？」
- 自分も同じだったという共感

### 2. 導入（300-400字）
- 自分の体験への橋渡し
- 「僕も昔は〜だった」
- 時代や状況の設定

### 3. 本文（2000-2500字）
- **ストーリーパート**: 何が起きたか、時系列で
- **構造パート**: なぜそうなったか、ロジカルに分析
- **再現性パート**: 読者が同じことをするには？

### 4. まとめ（400-500字）
- Solution（解決策）の要約
- 読者への問いかけ・CTA
- 次回予告やシリーズへの誘導

## 見出しルール
- H2は3-5個
- キャッチーに（「〜の3つの理由」より「〜してみた結果」）
- 数字を入れる（「3ヶ月で」「100万円」）

## 必須要素
- [ ] 具体的な数字が3つ以上
- [ ] 自分の失敗談が1つ以上
- [ ] 読者への問いかけが2つ以上
- [ ] 【要追記】マークで一次情報の追記箇所を明示
```

### 3.4 .claude/skills/generate-draft.md（下書き生成スキル）

```markdown
# スキル: 下書き生成

## 使い方
```
claude "記事XXXの下書きを生成して"
```

## 手順
1. `backlog/article-ideas.md` から該当記事の情報を取得
2. `context/story-bank.md` から関連エピソードを抽出
3. `.claude/rules/writing-style.md` の文体に従って執筆
4. `.claude/rules/article-structure.md` の構成に従う
5. 出力先: `articles/XXX/draft.md`

## 出力フォーマット
```markdown
# {タイトル}

{リード文: 読者のPainを突く}

## {見出し1}

{本文}

【要追記: ここに〜の具体的なエピソードを追加】

## {見出し2}

...

## まとめ

{Solution + CTA}
```

## チェックリスト
生成後、以下を確認:
- [ ] 文字数: 3000-4000字
- [ ] 【要追記】マークが3箇所以内
- [ ] 具体的な数字が3つ以上含まれている
- [ ] 文体が writing-style.md に沿っている
```

### 3.5 .claude/skills/extract-x-posts.md（X投稿抽出スキル）

```markdown
# スキル: X投稿抽出

## 使い方
```
claude "記事XXXからX投稿を抽出して"
```

## 手順
1. `articles/XXX/final.md` を読み込む
2. 5タイプ × 2投稿 = 10投稿を生成
3. 出力先: `articles/XXX/x-posts.md`

## 投稿タイプ

### Type 1: 刺さるワンライナー（2投稿）
- 記事内の印象的な一文を抜粋・調整
- 140字以内
- 例: 「関東全ての自治体に電話した。お宝物件はネットにはない。」

### Type 2: 数字フック（2投稿）
- 具体的な数字を含む
- 例: 「26歳、貯金ほぼゼロ、信用もない。あったのは『何となくの自信』だけ。」

### Type 3: 問いかけ（2投稿）
- 読者への質問形式
- 例: 「物件探し、ネットだけで終わらせてない？」

### Type 4: 逆張り（2投稿）
- 常識への反論
- 例: 「最初に建築事業を選ぶスタートアップはない、と言われた。だから選んだ。」

### Type 5: 記事リンク付き（2投稿）
- 記事公開時用
- 要約 + URL
- 例: 「【新記事】26歳で古民家ホテルを作った話。物件探しから開業まで全部書いた。 {URL}」

## 5. 日常の執筆ワークフロー

### 5.1 週次サイクル（週3記事 × 6時間）

```
月曜: 記事A（2時間）
水曜: 記事B（2時間）
金曜: 記事C（2時間）
```

### 5.2 1記事の流れ（2時間）- 対話型

```
# ========== Phase 1: 対話 & 下書き生成（30分）==========

$ claude "物件探しの試行錯誤について記事を書きたい"

→ Claudeが深掘り質問（3-5問）
→ あなたが回答
→ Claudeが構成提案
→ 「OK」と返す
→ 下書き生成 + story-bank追記案

# ========== Phase 2: 編集（50分）==========

# draft.mdを開いて編集
$ code articles/20260115-property-search/draft.md

# 【要追記】マークの箇所に詳細を追加
# 事実確認、トーン調整

# ========== Phase 3: 校閲 & X投稿（20分）==========

$ claude "この記事を校閲して"
$ claude "X投稿も作って"

# ========== Phase 4: 公開（20分）==========

# Note: final.mdをコピペ → 公開
# X: x-posts.mdから投稿をスケジュール
```

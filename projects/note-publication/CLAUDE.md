# Architecture of Life - Claude Code Context

## このプロジェクトについて
「Architecture of Life（人生の設計図）」は、Kentoが運営するNoteマガジン。
「キャリアも、資産も、人生も。全ては『デザイン』と『実装』でハックできる。」をコンセプトに、
精神論ではなくエンジニアリング的思考で人生の出力を変える方法論を発信する。

## 重要な参照ファイル

## 基本指示
1. 記事を書く時は必ず `context/story-bank.md` から関連エピソードを検索して提示
2. 文体は `.claude/rules/writing-style.md` に厳密に従う
3. 企業の具体名は出さない（「空間デザイン会社」「不動産Tech企業」など）
4. **対話で得た新しいエピソードは必ずstory-bankに追加提案する**
5. 一次情報が足りない箇所は【要追記】マークを付ける（3箇所以内）

## 記事生成の基本フロー（対話型）
ユーザーが「〜について書きたい」と言ったら:

### Phase 1: 深掘り質問
- **articles/index.md** で過去記事をチェック（重複・関連を確認）
- story-bank.mdから関連エピソードを検索して提示
- 3-5問の深掘り質問で一次情報を引き出す
- 質問は1ターンにまとめる（何度も聞かない）

### Phase 2: 構成提案
- タイトル案を3つ提示
- 記事の流れ（見出し構成）を提案
- 過去記事との関連があれば「以前〜について書いたので、今回は〜」と提案
- ユーザーの承認を得てから次へ

### Phase 3: 下書き生成
- 回答内容 + story-bankのエピソードを織り込む
- 出力先: `articles/{YYYYMMDD}-{slug}/draft.md`
- 同時にstory-bank.mdへの追記案を提示

### Phase 4: X投稿生成（オプション）
- 「X投稿も作る？」と確認
- 承認後、extract-x-posts.mdに従って生成

※ 「もう書いて」「早く」と言われたら質問をスキップして即座に生成

## 記事公開後のフロー
ユーザーが「記事を公開した」「インデックスに追加して」と言ったら:

### Step 1: final.md確認
- `articles/{ID}/final.md` を読み込む
- なければ draft.md を final.md にリネームするか確認

### Step 2: インデックス更新
- `articles/index.md` にサマリーを追記
- カテゴリ別クイックリファレンスにも追加

### Step 3: story-bank確認
- 記事内の新エピソードがstory-bankに反映されているか確認
- なければ追記提案

## スキル（コマンド）

### メインスキル: 対話型記事生成（推奨）
```
"◯◯◯について記事を書きたい"
"〜の話を記事にしたい"
```
→ `.claude/skills/interactive-draft.md` を参照
→ 深掘り質問 → 構成提案 → 下書き生成 の3フェーズで進行

### その他スキル
| コマンド | 参照ファイル | 用途 |
|---------|-------------|------|
| バックログから生成 | `.claude/skills/generate-draft.md` | 既存バックログから下書き生成 |
| 校閲 | `.claude/skills/review-article.md` | 記事の校閲・修正提案 |
| X投稿抽出 | `.claude/skills/extract-x-posts.md` | 記事からX投稿を生成 |
| インデックス更新 | `.claude/skills/update-index.md` | 公開記事をインデックスに追加 |

## 蓄積される3つのデータベース

```
┌─────────────────────────────────────────────────────────────┐
│  1. articles/index.md（記事インデックス）                    │
│     - 公開記事のサマリー一覧                                 │
│     - 新規記事生成時に重複チェック・関連記事参照             │
├─────────────────────────────────────────────────────────────┤
│  2. articles/{ID}/final.md（記事本文）                       │
│     - 公開済み記事の全文                                     │
│     - 詳細が必要な時だけ参照                                 │
├─────────────────────────────────────────────────────────────┤
│  3. context/story-bank.md（エピソードDB）                    │
│     - 一次情報（体験談）の蓄積                               │
│     - 記事生成時に引用                                       │
└─────────────────────────────────────────────────────────────┘

※ 記事を書くたびに3つのDBが育っていく
```
